## 三、怎么赚钱（价值主张 + 定价）

### 双引擎变现设计

```
                ┌─────────────────┐
                │   🏭 收入主轴    │
                │  酷家乐效果图接单 │
                │  月目标：3000 元  │
                └────────┬────────┘
                         │
                ┌─────────────────┐
                │   🌸 收入辅轴    │
                │  小红书代发商品   │
                │  月目标：3000 元  │
                └────────┬────────┘
                         │
                ┌─────────────────┐
                │   📺 信任层      │
                │  B站 UP 主        │
                │  3900 → 10 万粉  │
                │  （过程即内容）  │
                └─────────────────┘
```

**设计逻辑**：
- 主轴（酷家乐）是**即时变现**——有接单即有收入，第 1 周可以开始
- 辅轴（小红书代发）是**流量变现**——需要积累粉丝，4-8 周见效
- 信任层（B站）是**长期壁垒**——开发过程本身就是内容，B站粉丝信任 → 转化购买工具/服务

### 定价策略（两阶段）

| 阶段 | 产品 | 价格 | 时机 |
|------|------|------|------|
| **阶段 A** | 工具一次性买断 | **199 元** | 先起量，降低用户决策成本 |
| **阶段 B** | SaaS 订阅制 | **39 元/月** | 用户粘性建立后切换 |

**定价理由**：先 A 后 B，先验证愿意为工具付费，再转向稳定现金流模式。

## 四、商业模式（Lean Canvas 摘要）

| 模块 | 内容 |
|------|------|
| **问题** | 设计接单靠人肉分散，没有时间追踪；小红书选品完全靠感觉，无数据支撑 |
| **客群** | 兼职/全职接单的室内设计师 + 想做代发的小红书新人 |
| **独特价值** | 自用验证的工具（不是外行写的功能堆砌），且自己就是第一个用户 |
| **解法** | 酷家乐时间记录工具 + 小红书 AI 选品 + 笔记生成工具 |
| **渠道** | B站过程记录 → 自然流量；小红书运营 → 代发商品流量 |
| **收入** | 工具买断（199） + 订阅（39/月） + 小红书代发佣金 |
| **成本** | API 费用（DeepSeek 便宜）+ 时间（最大成本）|
| **关键指标** | 酷家乐月接单数 / 小红书万粉里程碑 / B站粉丝增长 |
| **护城河** | 个人品牌（B站+小红书）+ 数据飞轮（自用积累的使用数据）|

## 五、MVP 验证计划（两条线同步）

### 酷家乐 MVP（4 周）

| 周 | 目标 | 核心行动 |
|----|------|---------|
| **第 1 周** | 时间审计 | 接 3-5 单，用时间记录工具（已上线），摸清每单实际耗时 |
| **第 2-3 周** | 提速实验 | 用 AI 辅助需求确认，看能否减少沟通往返次数 |
| **第 4 周** | 规模化测试 | 冲 10 单，看时间和精力上限在哪 |
| **验收** | 月 20 单 3000 元 | 时薪 > 100 元算合格 |

### 小红书 MVP（4 周）

| 周 | 目标 | 核心行动 |
|----|------|---------|
| **第 1 周** | 选品 + 建号 | AI 工具分析爆款，挑 3-5 个测试商品 |
| **第 2-3 周** | 内容测试 | 发 10-20 篇笔记，看哪种内容起量 |
| **第 4 周** | 流量变现 | 有流量的方向加倍，没流量的停掉 |
| **验收** | 一个月 3000 佣金 | 数据不满足 → 重选赛道，不死磕 |

## 七、安全网设计（风险隔离）

**核心逻辑**：辅轴失败不影响主轴。

```
主轴（酷家乐接单）= 保底收入，失败成本低（已有技能，无需额外投入）
辅轴（小红书代发）= 高风险高回报，4周不过就换方向
信任层（B站）= 长线布局，不以短期变现为目标
工具（Rubedo等）= 自用赚钱，开放是锦上添花
```

**止损原则**：
1. 4 周后辅轴没有正收入信号 → 换选品赛道，不换平台
2. 3 个月后主轴收入不达 2000 → 反思时间利用，不扩线
3. 工具开发不能占超过主业工作日时间的 30%

### 战略一致性声明

本白皮书（v1.0）定位为"自用工具的独立开发者"，核心是先用工具赚钱、再迭代工具。本次校准在此基础上补充了**方法论的来源**——赚钱的方法从哪里来？答案是：**从已经成功的人那里学**。Nigredo 和 Albedo 就是"学习+验证"的加速器。

三条线的关系：
1. **赚钱线**（酷家乐 + 小红书）= 立即产生现金流，是生存基础
2. **学习线**（Nigredo + Albedo）= 提高赚钱效率，找到更好的赚钱方法
3. **中枢线**（Citrinitas）= 把学到的知识和执行的数据都存下来，让下一次模仿更聪明
