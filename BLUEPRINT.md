# ⚛️ Opus Magnum

> *Verum sine mendacio, certum et verissimum.*  
> *真实不虚，确然无疑。* — 翠玉录 (Tabula Smaragdina)

---

## 🎯 愿景

**一人即公司，个体即超级。**

一个普通人，借助 AI 工具链，完成从「信息摄入 → 知识提炼 → 真伪验证 → 内容创作 → 多平台分发 → 私域经营」的完整闭环。

如同炼金术士将铅转化为黄金——我们将互联网上的噪音转化为可盈利的知识资产。

---

## 🏛️ 命名体系：炼金术士的工坊

炼金术的核心隐喻完美契合本项目的本质：

| 阶段 | 炼金术概念 | 含义 | 对应 |
|------|-----------|------|------|
| **Nigredo** 黑化 | 分解、腐败、回归混沌 | 将原始内容分解为基本单元 | 数据摄入 |
| **Albedo** 白化 | 洗涤、净化、去除杂质 | 清洗数据、提取真知 | 知识提取 |
| **Citrinitas** 黄化 | 转化、觉醒、内在之光 | LLM 理解、结构化 | 知识转化 |
| **Rubedo** 红化 | 完成、哲人石诞生 | 最终产出、变现 | 内容输出 |

每个子项目以**炼金术工具/概念**命名，合在一起构成完整的「巨作」(Magnum Opus)。

### 🏭 Athanor · 熔知 / MindForge — 知识引擎

> *Athanor 是炼金术士的自给熔炉，一旦点燃永不熄灭。*

| 属性 | 值 |
|------|-----|
| **旧名** | 知炬 / KnowledgeForge |
| **功能** | 知识库的存储、检索、管理 |
| **输入** | Markdown文档、网页文章、对话记录、视频字幕 |
| **核心** | Qdrant 向量库 + Ollama 嵌入 + LLM 问答 |
| **状态** | 🔨 已完成 MVP (Phase 3 多页面架构) |

**地位**: 整个系统的中央知识库。所有其他工具产出的知识最终流入 Athanor。

## 🛡️ 护城河（竞争壁垒）

### 1. 数据飞轮
每个子项目不是孤立的——它们互相喂养：
- Alembic 产出的文档 → 存入 Athanor 增强检索
- Crucible 的验证结果 → 提升 Athanor 内容质量
- Athanor 的知识 → 供给 Elixir 生成高质量内容
- Elixir 发布的反馈数据 → 回流优化 Alembic 的分析

竞争对手可以复制一个工具，但无法复制整个飞轮。

### 2. 个人知识图谱
系统积累的不是通用数据，而是**你**的阅读习惯、兴趣领域、验证标准。
- 你的知识库结构是独一无二的
- 你的矛盾检测标准是你亲自校准的
- 你的内容风格模型是你在使用中训练出来的

### 3. 跨源矛盾检测
目前市面上没有任何产品能做到「跨B站视频 + 个人知识库 + 网页文章」的统一矛盾检测。这是一个技术壁垒。

### 4. 端到端闭环
从摄入到变现是完整的，不是拼凑的工具链。
- 市面上的替代方案：Notion + ChatGPT + 手动下载 + 手动分析
- Opus：一条自动化管道

### 5. 开源但不可复制
- 代码开源供参考
- 但具体配置、Prompt模板、个人知识库、调优参数是私有的
- 架构理解不难，积累到同等量级需要时间

### 6. 命名与品牌壁垒
炼金术命名体系天然具有辨识度和故事性：
- 搜索 "Athanor" 不会被淹没在通用词中
- 五器工坊的概念可以注册商标
- Logo 体系可以围绕炼金术符号构建（🜂 🜃 🜁 🜄 等 Unicode 符号）

## 🔎 GitHub 同类项目竞品分析

> *知己知彼，百战不殆。以下为 2026.06 调研结果。*

### 全景对比

| 项目 | Stars | 覆盖范围 | 与 Opus 对比 |
|------|-------|---------|-------------|
| **dreammis/social-auto-upload** | ⭐9.5K | 多平台视频发布（B站/抖音/TikTok/小红书/快手/视频号） | 🔶 做了 Elixir 的一半（只发布，不创作） |
| **cyfyifanchen/one-person-company** | ⭐2.7K | 一人公司工具推荐（LLM/设计/代码/生产力） | 🔶 资源列表，不是蓝图。我们的"怎么做"，它只是"用什么" |
| **OrangeViolin/content-pipeline** | 新兴 | AI内容管道：一个prompt→多平台发布 | 🔶 概念相似！但没有知识库支撑、没有验证层、面向纯内容分发 |
| **Libr-AI/OpenFactVerification (LOKI)** | ⭐1K+ | 事实核查五步管线 | 🔶 做了 Crucible 的一部分（声明提取+验证），但不支持中文、非跨文档 |
| **IshanRai9/Document-Semantic-Contradiction-Detector** | ⭐0 | 跨文档矛盾检测 7 步 NLI 管道 | 🔶 架构可借鉴，但不支持中文、无 D-S 融合 |
| **bilibili-api-python** | ⭐高 | B站全量 API 封装 | 🟢 可作为 Alembic 的数据采集轮子 |

### 关键发现

> 🎯 **目前没有项目覆盖「摄入→验证→存储→创作→分发」的完整闭环。**
>
> social-auto-upload 是 Elixir 最直接的竞品，但它只做了「分发」这一步，前面的「从什么内容分发」完全不管。  
> content-pipeline 概念最近似，但缺乏知识库和验证层，本质是一个 Claude Code Skill。  
> Opus 的护城河在于**五器联动**——单个工具可以被替代，整个飞轮难以复制。

### 可借用的轮子

| 轮子 | 用途 | 对应子项目 |
|------|------|-----------|
| `dreammis/social-auto-upload` | 多平台发布引擎（Playwright+API） | ✨ Elixir |
| `bilibili-api-python` | B站数据采集（弹幕/评论/UP主） | ⚗️ Alembic |
| `biliup-rs` | B站视频上传 CLI（Rust） | ✨ Elixir |
| LOKI 声明提取管线 | 声明提取 + 核查价值评估 | 🔬 Crucible |
| Document Contradiction NLI | 跨文档矛盾检测架构参考 | 🔬 Crucible |
