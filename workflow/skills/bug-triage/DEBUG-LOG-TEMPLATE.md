# 错误记录文件模板 (BUGLOG)

> 疑难 Bug 多轮未修复时，按此模板建一个临时记录文件，每次失败追加一条。
> 位置：`<项目>/.workbuddy/memory/BUGLOG-<主题>-<MMDD>.md`
> 解决后精简为 `BUGLOG-<主题>-<MMDD>-FIXED.md`，**双写归档**（见 bug-triage SKILL.md「归档位置」）：
>   ① 项目文件夹 `<项目>/.workbuddy/memory/`  ② 巨作统一管理 `D:/opus-magnum/docs/bug-triage-cases/`

---

## 文件头（置顶两块，每次动笔前先看）

### 已排除清单（永不重试这些方向）
- [ ] 方向A — 证伪于第N轮（一句话根因）
- [ ] 方向B — 证伪于第N轮

### 假设银行（候选根因，标状态）
| # | 假设根因 | 状态 | 验证轮次 |
|---|---------|------|---------|
| 1 | ... | 未验证 | — |
| 2 | ... | 已排除 | 第2轮 |
| 3 | ... | 已确认 | 第5轮 |

状态：未验证 / 验证中 / 已确认 / 已排除

---

## 每次失败追加一条（严格按字段）

### 第 N 轮 — YYYY-MM-DD HH:MM
- **假设方向**：这一轮想验证什么（对应假设银行哪一条）
- **改动点**：改了哪个文件 / 哪个函数 / 关键 diff 摘要（只改 1 个变量）
- **诊断快照**：粘贴诊断工具的一键复制输出（纯文本，不是截图）
- **结果**：通过 / 未通过 / 部分
- **失败根因**：为什么没生效（基于诊断数据，不是猜）
- **状态**：继续同方向 / 已排除(加入上方清单) / 待验证
- **下一步**：下一轮验证假设银行哪一条

---

## 收敛时（修复成功后）精简为 -FIXED 归档

```markdown
# BUGLOG-<主题>-<MMDD>-FIXED

> 元数据：项目 / 跨度 / 总轮数 / 触发方式 / 最终提交 / 流程依据
> 一句话背景：[这到底怎么回事]

## 1. 现象
- 技术表现：[精确现象 + 复现条件]
- 用户原话：「…」（若有）

## 2. 调试历程（证据链，按阶段分组；每阶段保留 假设→验证→诊断证据→结论）
### 阶段一：[方向名]（第 1~N 轮）
- 假设：[我们认为什么能解决]
- 验证方法：[改了什么 / 用什么诊断量的]
- 诊断证据：[粘贴当时的真实数值，如「日历高度恒 580px」]
- 结论：❌ 排除 — [证据说明的根因]

### 阶段 N：[最终方向]（第 X~Y 轮）✅
- 假设：[最终思路]
- 验证方法：[诊断量了什么]
- 诊断证据：[窗口态=… / 最大化态=… / 空白=…]
- 结论：✅ 确认

## 3. 最终方案
- 关键改动：[文件 + 改了什么 + 为什么有效]
- 验证：诊断达标（见阶段 N 证据）

## 4. 已排除方向（含证伪证据）
| # | 方向 | 轮次 | 诊断证伪证据 | 根因 |
|---|------|------|------------|------|
| 1 | [方向] | 1~10 | [当时诊断数值] | [根因] |

## 5. 教训（每条绑定具体证据）
1. [戒律]（源于阶段 X：诊断显示 Y）
```

> 归档即终点：归档文件本身就是沉淀，不再修改 bug-triage 的 skill 文件（SKILL 文件保持固定，详见 SKILL.md）。
