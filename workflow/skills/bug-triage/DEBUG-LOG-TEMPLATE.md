# 错误记录文件模板 (BUGLOG)

> 疑难 Bug 多轮未修复时，按此模板建一个临时记录文件，每次失败追加一条。
> 位置：`<项目>/.workbuddy/memory/BUGLOG-<主题>-<MMDD>.md`
> 解决后精简为 `BUGLOG-<主题>-<MMDD>-FIXED.md` 归档（同目录）。

---

## 文件头（置顶两块，每次动笔前先看）

### 已排除清单（永不重试这些方向）
- [ ] 方向A — 证伪于第N轮（一句话根因）
- [ ] 方向B — 证伪于第N轮

### 假设银行（候选根因，标状态）
| # | 假设根因 | 状态 | 验证轮次 |
|---|---------|------|---------|
| 1 | ... | 未验证 | — |
| 2 | ... | 已排除 | 第2轮 |
| 3 | ... | 已确认 | 第5轮 |

状态：未验证 / 验证中 / 已确认 / 已排除

---

## 每次失败追加一条（严格按字段）

### 第 N 轮 — YYYY-MM-DD HH:MM
- **假设方向**：这一轮想验证什么（对应假设银行哪一条）
- **改动点**：改了哪个文件 / 哪个函数 / 关键 diff 摘要（只改 1 个变量）
- **诊断快照**：粘贴诊断工具的一键复制输出（纯文本，不是截图）
- **结果**：通过 / 未通过 / 部分
- **失败根因**：为什么没生效（基于诊断数据，不是猜）
- **状态**：继续同方向 / 已排除(加入上方清单) / 待验证
- **下一步**：下一轮验证假设银行哪一条

---

## 收敛时（修复成功后）精简为 -FIXED 归档

```markdown
# BUGLOG-<主题>-<MMDD>-FIXED

## 症状
[一句话：什么操作下出现什么现象]

## 最终根因
[真正的原因，一句]

## 最终方案
[关键改动：文件 + 改了什么 + 为什么有效]

## 教训（可复用戒律）
[一条通用经验，未来类似场景直接用]

## 已排除方向
- 方向A（×轮）、方向B（×轮）...
```

> 归档后：把「教训」提练进 bug-triage 的 `ANTI-PATTERNS.md` 或你的长期记忆，让下次直接跳过这些坑。
