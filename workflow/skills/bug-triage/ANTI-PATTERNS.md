# 反模式警示 (ANTI-PATTERNS)

> 从真实疑难 Bug（如凝华日历底部空白，19 轮修复）提炼的通用戒律。卡住时对照检查。

---

## 1. 同一类方案反复试
同方向连续失败 ≥3 次还在试同一个思路 = 违反"重试≤3"。第 3 次失败 MUST 换方案或停下来问用户。
凝华案例：改 DayPilot 高度的 10 次尝试全败，应更早转向。

## 2. 猜第三方库内部逻辑
不要猜库的内部机制，去读文档 + 量真实 DOM（如 `getBoundingClientRect`）拿证据。
凝华案例：以为改 `height` 能撑高网格，实际网格高度由 `cellHeight` 决定；`height:"auto"` 只在 init 算一次。

## 3. 用错测量基准
别用「容器高 − 渲染高」猜空白，测「真实边缘差」（如日历底边缘 − 面板顶）。
被 CSS `!important` 钉死时，差值永远≈0，永远测不到真空白。

## 4. CSS `!important` 优先级陷阱
内联样式的 `!important` 优先级高于 CSS 文件的 `!important`。想覆盖第三方库内联样式，得用同级别或更具体的选择器 + `!important`，或改它的配置而非对抗样式。
凝华案例：DayPilot 写 `#calendar { height:580px !important }`，CSS 的 `!important` 干不过它。

## 5. 全屏/绝对定位忘了减导航栏高
`height:100vh` 的布局没减顶部导航栏高度 → 整体溢出视口 → 底部被裁到屏外（看似"还有空白"其实是溢出）。
凝华案例：溢出 84px 导致底部面板底边被裁。

## 6. 测量竞态 / 振荡
resize 后第三方库异步重排 DOM，JS 测旧位置盖过去、库又推下 → 残留空白反复。
当测量引发竞态/振荡时，**锁定固定常量优于反复测量**。

## 7. 验证走控制台日志
不要用 `console.log` 验证，让用户开 F12 看密密麻麻的日志。用应用内诊断工具（🔧 按钮等），输出一键复制纯文本。
凝华案例：用户明确否决 console.log，改用诊断按钮。

## 8. 窗口态/最大化态高度不同
面板高度跟着窗口变（窗口 280 / 最大化 332）→ 两种状态空白大小不同，用户感知"不一致"。
解法：把会随状态变的高度锁成固定常量（如 `--panel-h:360px`），两态一致。

## 9. 工程坑：误建保留设备名文件
沙箱里 `> nul` 重定向可能误建名为 `nul` 的 Windows 保留设备名文件，导致 `git add -A` 直接报错。
遇到 `git add` 报 nul 错误：删掉该文件 + `.gitignore` 加 `nul` / `*.lock` / 日志。
